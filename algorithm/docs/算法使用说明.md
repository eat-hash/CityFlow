# 交通信号配时算法使用说明

## 1 项目概述

### 1.1 项目简介
本项目基于 **Webster 经典交通信号配时算法** 开发，实现城市路口信号灯自动配时功能。  
支持单路口计算、多路口批量并行计算，可直接解析路网、车辆轨迹 JSON 数据集，从原始仿真数据自动统计车流量，完成信号周期、相位绿灯时长、车辆平均延误、路口通行能力一体化计算。

### 1.2 核心功能
- 解析路网、车辆轨迹 JSON 文件，自动统计各路口四相位车流量；
- 基于 Webster 算法计算最优信号周期、各相位绿灯时长；
- 输出车辆平均延误、路口通行能力两项核心评价指标；
- 完善参数校验、异常捕获，拦截非法参数、全零流量等异常场景；
- 基于线程池实现批量并行计算，保证大批量路口运算性能；
- 统一前后端交互数据格式，接口规范清晰。

### 1.3 相位定义（全局统一）
系统固定划分四个通行相位，全局关键字统一：

| 相位名称 | 说明 |
| --- | --- |
| 东西直行 | 东西方向直行车流 |
| 东西左转 | 东西方向左转车流 |
| 南北直行 | 南北方向直行车流 |
| 南北左转 | 南北方向左转车流 |

## 6 算法原理（Webster 信号配时）

### 6.1 整体计算流程
1. 读取各相位车流量，计算相位流量比与总流量比；
2. 校验总流量，全零流量直接抛出异常，终止计算；
3. 使用 Webster 公式计算理论最优周期，并结合最小 / 最大周期做边界约束；
4. 计算有效绿灯总时长，按各相位流量比例分配绿灯，同时保证不低于最小绿灯；
5. 修正绿灯时长误差；
6. 计算车辆平均延误、路口通行能力两个评价指标。

### 6.2 核心计算公式

**最优信号周期**
\[
C_0 = \frac{1.5L + 5}{1 - Y}
\]
- \(L\)：所有相位总损失时间
- \(Y\)：所有相位流量比之和

**Webster 车辆平均延误**
\[
d = \frac{C(1-Y)}{2(1-Y)} + \frac{X^2}{2q(1-X)}
\]

**通行能力**
基于饱和流量、有效绿灯占比换算为百分比，上限限制为 100%。

## 7 测试用例说明
`AlgorithmTest` 内置 7 组测试场景，覆盖功能、边界、异常、性能、真实数据：
- 常规流量测试：标准四相位车流，验证正常配时逻辑；
- 边界流量测试：仅单个相位有车流，验证极端分配规则；
- 零流量异常：所有相位流量为 0，验证异常拦截；
- 非法参数测试：最小绿灯超出范围，验证参数校验；
- 小规模批量：3 个路口批量计算，验证批量接口；
- 性能测试：10 个路口并行计算，要求耗时 < 1000ms；
- 真实数据集：加载本地路网 + 车辆数据，全量计算 32 个路口配时方案。

## 8 常见问题与解决

| 问题 | 原因 | 解决 |
| --- | --- | --- |
| 提示找不到 JSON 文件 | data 目录位置错误 / 文件名不匹配 | 保证 data 文件夹与源码同级，文件名一致 |
| 编译提示找不到 Jackson 类 | 未导入 Jackson 依赖包 | 重新将 lib 下三个 Jar 包添加为项目库 |
| 所有路口配时完全一致 | 批量接口共用同一套基础流量 | 已优化为逐个路口独立使用自身流量 |
| 通行能力显示 100% | 路口车流量达到饱和状态 | 调大 maxCycle 或降低流量 |
| 平均延误数值固定不变 | 附加延误趋近于 0 | 修改基准延误或使用更高流量数据集 |
