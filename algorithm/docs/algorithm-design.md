# 交通信号配时优化算法设计文档
## 一、算法名称
基于流量绿信比分配的单路口信号配时优化算法

## 二、输入数据
1. roadnet_4_4.json：路口拓扑、相位结构
2. anon_xxx.json：杭州真实交通流数据

## 三、输入格式
- intersectionId：路口ID
- phaseFlow：四相位车流量
- minGreen：最小绿灯
- maxCycle：最大周期

## 四、输出格式
{
"intersection_id": "路口ID",
"cycle_length": 周期,
"yellow_time": 黄灯时长,
"timing_plan": 各相位绿灯时长,
"average_delay": 平均延误,
"capacity": 通行能力
}

## 五、算法步骤
1. 读取真实JSON路网与流量数据
2. 按流量占比分配绿灯
3. 约束最小绿灯与最大周期
4. 输出配时方案与评价指标

## 六、服务封装与异常处理
1.  封装了 `AlgorithmService` 类，提供 `optimizeSingle` 和 `optimizeBatch` 两个对外接口
2.  前置校验：对空输入、非法参数（如最大周期过小）进行拦截
3.  异常捕获：对算法执行过程中的所有异常进行捕获，返回统一的 `error` 状态和错误信息

## 七、单元测试结果
| 测试用例 | 预期结果 | 实际结果 |
| --- | --- | --- |
| 正常流量路口 | 返回合理配时方案 | 周期110s，配时按流量分配 |
| 批量路口优化 | 返回3个路口的配时方案 | 全部正常输出 |
| 流量全为0 | 返回success，平均分配绿灯 | 周期110s，每个相位绿灯17s |
| 最大周期30s（非法） | 返回error，提示周期过小 | 状态error，提示“无法满足最小绿灯时长” |
